## 1. 按钮组件基础

### 1.1 Button 组件 (`components/ui/button.tsx`)

**基础按钮组件**，支持以下变体：

- `default`: 默认主色按钮
- `destructive`: 危险操作按钮（红色）
- `outline`: 边框按钮
- `secondary`: 次要按钮
- `ghost`: 幽灵按钮（无背景）
- `link`: 链接样式按钮

**尺寸**:
- `sm`: 小尺寸
- `default`: 默认尺寸
- `lg`: 大尺寸
- `icon`: 图标按钮

### 1.2 AsyncButton 组件 (`components/ui/async-button.tsx`)

**异步按钮组件**，自动处理加载态、错误提示、防抖。

**变体**:
- `primary`: 主色渐变按钮
- `secondary`: 次要按钮
- `danger`: 危险操作按钮

## 🎨 按钮样式规范

### 按钮变体使用规范

1. **default (主色按钮)**
   - 用于主要操作：提交、确认、保存
   - 示例：提交申请、确认付款

2. **destructive (危险按钮)**
   - 用于危险操作：删除、拒绝、取消
   - 示例：删除商品、拒绝申请

3. **outline (边框按钮)**
   - 用于次要操作：取消、返回、查看
   - 示例：取消操作、返回列表

4. **ghost (幽灵按钮)**
   - 用于不重要的操作：删除、编辑
   - 示例：删除购物车商品

5. **secondary (次要按钮)**
   - 用于次要但重要的操作
   - 示例：查看详情、导出数据

6. **link (链接按钮)**
   - 用于文本链接样式的操作
   - 示例：了解更多、查看全部

### 按钮尺寸使用规范

- **sm**: 用于表格、列表中的操作按钮
- **default**: 用于表单、卡片中的主要按钮
- **lg**: 用于页面顶部、重要操作按钮
- **icon**: 用于图标按钮、工具栏按钮

## 📝 注意事项

1. **按钮状态管理**
   - 所有异步操作按钮应使用 `AsyncButton` 组件
   - 按钮应正确显示加载状态
   - 按钮应正确处理禁用状态

2. **按钮权限控制**
   - 按钮应根据用户权限显示/隐藏
   - 使用 `hasPermission` 检查权限
   - 按钮操作应进行后端权限验证

3. **按钮交互反馈**
   - 按钮点击应有视觉反馈（动画、颜色变化）
   - 操作成功/失败应有提示（toast）
   - 危险操作应有确认对话框

4. **按钮可访问性**
   - 按钮应有明确的文本或图标
   - 按钮应有合适的尺寸（最小44x44px）
   - 按钮应有焦点状态
